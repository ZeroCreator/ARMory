# Планировщик задач

Модуль позволяет создавать задачи, привязанные к проектам и разделам, с датами, повторениями, напоминаниями и заметками.

## Включение / отключение

Чтобы полностью скрыть вкладку **«Планировщик»** из навигации, установите в `.env`:

```env
SCHEDULER_ENABLED=false
```

По умолчанию `SCHEDULER_ENABLED=true`. Отключение планировщика не влияет на раздел **«Календарь»** и вкладку **«Синхронизация»**.

## Структура задачи

- Название
- Описание
- Проект и раздел (опционально)
- Статус: `pending`, `in_progress`, `completed`, `cancelled`
- Приоритет: `low`, `medium`, `high`
- Дата и время выполнения
- Повторение: ежедневно, еженедельно, ежемесячно, ежегодно
- Напоминание: заранее от даты выполнения
- Заметки

## Повторяющиеся задачи

При создании задачи с повторением оригинальная задача остаётся, а при её выполнении система генерирует следующее вхождение с новой датой на основе выбранного правила.

## Уведомления

Напоминания можно получать:
- в интерфейсе (бейджи и всплывающие уведомления)
- через email (если настроен SMTP)

## API

Основные эндпоинты:

```http
GET    /scheduler/tasks
POST   /scheduler/tasks
GET    /scheduler/tasks/{id}
PUT    /scheduler/tasks/{id}
DELETE /scheduler/tasks/{id}
POST   /scheduler/tasks/{id}/complete
POST   /scheduler/tasks/{id}/snooze
```

## Календарь

Календарь доступен отдельно и показывает задачи, дедлайны и события по месяцам. Подробнее в [calendar.md](calendar.md).
