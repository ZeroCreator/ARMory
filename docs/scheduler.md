# Планировщик задач

Модуль на главной странице ставит shell-скрипты проектов в очередь на одноразовый запуск в заданное время. Выполнение — через системную утилиту `at`.

## Откуда берутся задачи

Планировщик читает переменную `SCRIPTS_PROJECT_PATHS` из `.env` — список каталогов проектов со скриптами через запятую (`~` разворачивается в `$HOME`):

```env
SCRIPTS_PROJECT_PATHS=~/scripts/ssk-scraper,~/scripts/ARMory
```

В `.env` каждого такого проекта задаётся словарь `SCHEDULER_TASKS`:

```env
SCHEDULER_TASKS='{"send-report": {"name": "Ежедневный отчёт", "script": "send_report.sh"}}'
```

В интерфейсе и API задача адресуется ключом `<проект>:<ключ>` — имя проекта (каталога) приводится к нижнему регистру, например `armory:send-report`.

## Удалённые серверы (прод, Docker)

На проде ARMory работает в Docker и сам ничего не выполняет: задачи ставятся на целевые серверы по SSH, где их исполняет локальный `at`. Это позволяет запускать скрипты в родном окружении каждого сервера, а очередь переживает редеплой контейнера.

Настройка:

1. Установите `at` на каждый сервер со скриптами (`apt install at`), убедитесь, что демон `atd` запущен.
2. Сгенерируйте выделенный ключ: `ssh-keygen -t ed25519 -f deploy/id_ed25519 -N ""`. Публичную часть добавьте в `~/.ssh/authorized_keys` пользователя на каждом сервере.
3. Заполните `.env`:

```env
# Пути к проектам — как они выглядят на целевых серверах
SCRIPTS_PROJECT_PATHS=/home/user/scripts/dcript,/home/user/flat-parser
# Привязка проектов к серверам (проект — имя каталога)
SCRIPTS_PROJECT_SSH=dcript=user@192.168.1.10,flat-parser=user@192.168.1.20
# Ключ: путь внутри контейнера и путь на хосте (монтируется в compose.yml)
SCHEDULER_SSH_KEY=/app/ssh/id_ed25519
SCHEDULER_SSH_KEY_HOST=./deploy/id_ed25519
```

Проекты без записи в `SCRIPTS_PROJECT_SSH` выполняются локально — режим разработки не меняется. Время выполнения трактуется в локальной таймзоне целевого сервера.

## Планирование

Во вкладке «Планировщик» выберите задачу и дату/время — скрипт будет поставлен в очередь `at`. Скрипт запускается из каталога своего проекта, поэтому относительные пути внутри него работают.

`POST /api/scheduler/schedule` дополнительно принимает массив `args` — аргументы добавляются к команде запуска скрипта:

```json
{
  "project": "armory:send-report",
  "datetime": "2026-07-20T09:00",
  "args": ["--mode", "full"]
}
```

## Очередь

- Список запланированных задач — `GET /api/scheduler/atq` (вывод `atq` со всех задействованных серверов; каждая строка снабжена префиксом `[сервер]` или `[local]`).
- Удаление из очереди — `POST /api/scheduler/remove-task` с `task_id`: просто `ID` для локальной очереди или `<сервер>:<ID>` для удалённой (выполняется `atrm`).

## API

```http
GET  /api/scheduler/tasks
POST /api/scheduler/schedule
GET  /api/scheduler/atq
POST /api/scheduler/remove-task
```
