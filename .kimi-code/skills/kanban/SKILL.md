Ты работаешь с канбаном ARMory. Аргумент `$task_ref` может быть:

- ссылкой вида `https://armory.team-73.ru/projects/{project_id}/kanban?task={task_id}` или `/projects/{project_id}/kanban?task={task_id}`;
- номером задачи вида `#39` или просто `39`;
- отсутствовать, если пользователь просит создать новую задачу.

## Получение задачи

Если `$task_ref` содержит номер или ссылку:

1. Извлеки `project_id` и `task_id`.
   - Для URL используй шаблон `/projects/{project_id}/kanban?task={task_id}`.
   - Для номера `#N` или `N` используй `task_id = N`.
   - Если `project_id` не указан, а речь идёт о проекте ARMory, используй `project_id = 14`.
2. Вызови MCP-инструмент `mcp__armory__get_task` с аргументами `project_id` и `task_id`.
3. Выведи краткую сводку: номер задачи, проект, название, статус, приоритет, теги, список, ответственного, дедлайн.
4. Все последующие изменения выполняй в контексте этой задачи.
5. Названия коммитов и веток желательно начинай с номера задачи, например `#39: fix deadline display`.

## Создание задачи

Если пользователь просит создать задачу:

1. Определи `project_id` (из контекста или спроси).
2. Получи список статусов проекта через `mcp__armory__list_tasks`, чтобы понять доступные `status_id`, или используй первый статус по умолчанию.
3. Вызови `mcp__armory__create_task` с заполненными полями: `status_id`, `title`, `description`, `priority`, `tags`, `list_name`.
4. Сообщи номер созданной задачи.

## Обновление задачи

Если пользователь просит изменить статус, название, описание, приоритет, теги, дедлайн или ответственного:

1. Определи `project_id` и `task_id`.
2. Вызови `mcp__armory__update_task` с только теми полями, которые изменились.
3. Подтверди результат.

## Не делай

- Не пытайся читать `data/armory.db` напрямую: основная база не локальная.
- Не хардкоди продакшен URL в коде или конфигах.
- Не создавай задачи без явного подтверждения пользователя, если не хватает обязательных полей.
